# Security Policy

## Reporting a vulnerability

Please do not report suspected vulnerabilities through a public issue.

Prefer one of these channels:

- GitHub private vulnerability reporting, if it is enabled for this repository
- direct private contact with the maintainer through the repository owner's
  public profile

When reporting an issue, include:

- affected files or commands
- a clear description of impact
- reproduction steps
- any mitigation you already tested

## Scope

Security issues include, for example:

- accidental exposure of operator state or recovery artifacts
- unsafe snapshot or restore behavior
- privilege, isolation, or data-leak problems in launch staging
- publication of private-only support-system surfaces on public `main`
